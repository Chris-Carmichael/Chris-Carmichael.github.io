Blogging With Hugo

September 5, 2019

Time to Blog

I've been into computers for over 30 years, since I was a teenager taking apart a Tandy 1000 EX. Yikes my mom was so mad. She threatened to take a broom to my good friend Jason and I if we didn't have it put back together by the morning. We succeeded, and I've been addicted to technology ever since.

Professionally, I've got ~20 years of experience. I recently had an opportunity to land an evangelist job talking about kubernetes, but the hiring manager couldn't find me online. He told the person connecting us “Dude, I can't find any footprint of your friend online. Are you sure this guy exists!?". It's a pretty funny story, but also a missed opportunity to advocate for a technology that will be more disruptive than VMware and OS virtualization in the years to come.

So, here I am, starting a blog. Hopefully it will help folks solve problems and prevent me from avoiding future missed opportunities. I also enjoy writing, so I'm excited about this new start.

Blogging with Hugo

Confession time: I've attempted to blog before. First Wordpress, then Google Sites, or whatever it was before that. It was not a pleasant experience. Clunky, insecure, and most importantly…no portability. I don't like being locked into a specific technology, especially for something as important as documentation and sharing knowledge. That, combined with the fact that I tend to get engrossed in projects I work on meant that internet blogging took a back seat.

I was at Jenkins World a few weeks back, and I saw a neat demo where a guy from CloudBees used a Jenkins X pipeline to publish blog posts written in Markdown, converted to a site using Hugo, and hosted it on Github Pages. I had looked at Hugo a few years back, and it didn't stick. This was when I was beginning my shift to DevOps, so Markdown felt a little clunky to me. Today, everything I write is Markdown, so I decided to give Hugo another try and it is surprisingly easy in 2019!

Setup Notes

I'm not doing anything as fancy as a Jenkins X pipeline with GitOps approval. Maybe I'll set that up and cover in a different post in the future. Here are some handy links I used with a brief outline.

This is also using Windows 10 as my work OS. I love Linux, but my background is Windows and I find the W10 + WSL combo to meet most of my needs. This also works well because some companies standardize and enforce Windows. I usually have a fleet of linux servers to support, so problem solving isn't an issue for me.

  • Create a private Github repo named blog

  • Create a public repo named Chris-Carmichael.github.io

  • Create a custom domain chris-carmichael.dev (I use NameCheap)

  • Add Github DNS ANAME records to www.chris-carmichael.dev DNS zone.

  • Plug in custom URL www.chris-carmichael on the github.io repo settings. Github partnered with LetsEncrypt in 2018 to automate the certificate management. Really slick.

Time for configuration:

# install hugo

choco install hugo -y

# clone the repos previously created

git clone git@github.com:Chris-Carmichael/blog.git
git clone git@github.com:Chris-Carmichael/Chris-Carmichael.github.io.git

# create the blog site and copy the contents to the blog repo

cd ~
hugo new site blog
cp -Recurse .\blog\* .\repos\blog\

# Check to make sure an empty page is up

hugo server -D

Ok, now you should be able to enter http://localhost:1313 on your browser of choice. It's not very pretty though, is it? Next I looked into Hugo Themes, did some research, played with a handful and settled on Hugo Universal Theme.

My main requirements were Disqus comments and search, and this theme gave me a landing page to showcase the skills and tools I enjoy working with professionally, and a handy blog link with search and comments. All good.

# add the theme

cd ~/repos/blog/themes
git clone https://github.com/devcows/hugo-universal-theme

# remove git references, as I will customize a bit.

rm -Recurse .\.git\ -Force

# create this post

hugo new content/blog/blogging-with-hugo.md

Publishing New Posts

Now that a post is ready to be published, here are the next steps.

  • Hugo generates the site to /blog/public

  • Contents of the /blog/public directory need to be copied to the Github repo

  • Commit and push the contents to Github

I automate everything I can, and this is no different:

# update blog

function Update-Blog {

    param (
        [Parameter(Mandatory=$true)]
        $comment
    )

    # save location, go to blog dir

    Push-Location
    cd ~/repos/blogging/blog

    # clear public folder, generate & update blog

    rm -Recurse C:\Users\carmi\repos\blogging\blog\public\ -Force
    hugo
    cp -Recurse .\public\* ..\Chris-Carmichael.github.io\ -Force
    cd C:\Users\carmi\repos\blogging\Chris-Carmichael.github.io
    git add .
    git commit -m $comment
    git push

    # reminder in case you don't see the posting on the interwebz

    Write-Output "Don't forget to set draft: false on your post"

    # go back to where you started

    Pop-Location

}

Set-Alias -Name blogupdate -Value Update-Blog

# preview blog

function Invoke-BlogDraft {

    # go to blogdir & preview

    cd ~/repos/blogging/blog
    hugo server -D

}

Set-Alias -Name blogpreview -Value Invoke-BlogDraft

Drop that in $profile and simply run blogpreview to take a look before you publish, then blogupdate to push the repo. Easy peasy.
